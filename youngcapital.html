July, 2017 - Jan, 2018
UX Design

Goal

The goal in this project is to improve the whole user journey from ‘candidates looking for jobs’ to ‘recommend jobs to candidates’, integrating the method of machine learning.

Deliverables are responsive website, including desktop, mobile, and tablet version, lanuched in 9 countries.

YoungCapital

As the most popular employment agency among young generation, YoungCapital has been active since 2000. With over 20 subsites under YoungCapital around Europe, design needs to be consistant for each country and scalable to different languages, while keeping the creativity of brand. Fight the best user flow against corporation legacy is part of the challenge.

Understand user journey

1

Gathering both qualitative and quantitative insights such as Hotjat, Google Analytics, Usabilla, user tests to find out where the problem is and which to start at first.

Architecture optimization

2

Started with structure optimization, features are extracted out and reorganized in more natural and logical way, making sure users find right information quickly.

Design

3

- Smarter service

The core value of YC is to help users find career opportunities in a satisfied way, hence design needs to be smart enough. Recommending user-wanted jobs intelligently, without them spending time on typing, manually searching for jobs is a key improvement.

This entails changes not only on interfaces, backend sysytem, but also the way how internal staff work.

- Design for right user segment

With a target group from 18 - 25 years old young generation, who are open for new gadget, design is set to be bold and outstanding. On the other hand, when designing for the client group with a higher average age, who are offering jobs on this platform and, clearness is of first priority.

- Improve efficieny and Satisfaction

Overlay is largely used through this project. Opening an overlay on top of the original page while browsing a sea of jobs reduces users’ fear of lost. This switch gets massive progress on efficiency and user satisfaction.

Validation

4

- Prototype

Interactive prototype is made in InVision along this project to get instant user feedbacks and iterates quickly. It also communicates function feasability with front-end and back-end developers.

- Tests

User tests are conducted from beginning till the end of launch and are still frequently updated. Over 10 types of method are applied based on different purposes, eg., scenario tasks, talk-out-loud, card sorting. Yet usability benchmarking is the mostly used for keeping track on how design performs on a weekly basis.
